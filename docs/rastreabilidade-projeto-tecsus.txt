Rastreabilidade Projeto Tecsus
Este documento define o sistema de rastreabilidade utilizado no projeto, descrevendo como solicitações, requisitos, tarefas, implementações e testes são relacionados ao longo do ciclo de desenvolvimento.

A rastreabilidade permite acompanhar a evolução do projeto desde a solicitação inicial até o deploy da funcionalidade, garantindo maior organização, transparência e facilidade de manutenção.

2. Convenção de Identificadores (IDs)
2.1 Objetivo
Os IDs são identificadores padronizados utilizados para representar os diferentes elementos do projeto, permitindo rastreabilidade entre artefatos como solicitações, requisitos, tarefas, testes e versões do sistema.

7. Convenção de Commits
7.1 Objetivo
O projeto utiliza o padrão Conventional Commits para padronizar o histórico de alterações, facilitar o code review e fortalecer a rastreabilidade entre requisitos, implementação e commits.

7.2 Estrutura
tipo[ID-task]: descrição da alteração

Exemplo: feat[SCRUM-04]: implementação do dashboard meteorológico

7.3 Prefixos
Prefixo
Significado
feat
nova funcionalidade
fix
correção de bug
refactor
melhoria de código
docs
alteração em documentação
style
alteração de estilo


8. Fluxo de Rastreabilidade
O fluxo descreve como os artefatos se relacionam ao longo do desenvolvimento:

Request → Requisito → Issue (Jira) → Branch → Commit → Merge Request → Pipeline CI → Code Review → Merge na Develop → Merge na Main → Deploy
